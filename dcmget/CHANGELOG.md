# DcmGet 版本说明

## 2.7.0（2026-07-17）

- 任务主页新增 PDI 快捷开关和输出目录选择，无需进入设置页即可快速启用并选择便携目录保存位置。
- PDI 中文 OHIF 阅片器默认单击选择序列，并默认显示患者四角标信息，优化日常阅片交互。
- 增加 DICOM 中文字符集容错和编码修复，改善患者姓名、检查与序列描述在 PDI 中的乱码问题。
- 换用全新 DcmGet 品牌图标，并优化高 DPI 窄窗口布局、大批量任务摘要以及阅片服务启动反馈。
- C-MOVE 结果会核对 PACS 完成子操作数与本地实际接收文件数，避免“PACS 报告完成但本地无文件”被误判成成功。
- `storescp` 接收器的 association abort 改为独立告警，不再无条件降级当前检查号状态；真实 DIMSE 失败仍会正确标记。
- 下载速度统计改为增量与自适应扫描，降低大量文件下频繁遍历目录的界面和磁盘开销。
- 研究匿名和严格匿名补充高风险自由文本清理，并覆盖嵌套序列中的敏感字段。

## 2.6.5（2026-07-17）

- 任务主页增加“打开已有 PDI 目录”：重启后可直接选择 U 盘或历史 PDI 根目录，程序记住上次位置，并只使用安装包内经过校验的 OHIF 与本地服务；新旧 PDI 都通过目录路由打开，用户不再需要寻找或选择 JSON 文件，也不会加载介质内可能被替换的网页脚本。
- 顶部状态与操作区改为稳定的双行布局，高 DPI 窄窗口不再挤压关键状态；禁用的主按钮会显示为明确的灰色状态，避免误判为可点击。
- C-MOVE 会解析最终 DIMSE 状态和完成、失败、警告子操作数；PACS 返回 `0xB000` 等部分失败时不再误报“完成”，而是保留已收文件并标记“部分成功”。
- PDI 取消改为后台幂等清理；Windows `taskkill` 增加超时和直接强制终止回退，停止或关闭时不再长时间卡住界面。
- DCMTK 部署缓存增加平台、版本、官方归档摘要及必需工具清单；每次复用都以固定 SHA-256 的 OFFIS 归档重新核对整个安装树，包含可执行文件、动态库、数据文件和链接，并拒绝缺失、篡改或额外注入的文件。Windows 构建同样要求保留并通过该官方归档认证。
- Windows 安装器将旧的固定 6666 防火墙规则迁移为仅限实际 `storescp.exe`、域/专用网络的程序规则，设置自定义接收端口后无需重新安装规则；升级仍保留原配置、注册码和试用状态。

## 2.6.4（2026-07-17）

- 主任务页增加高 DPI 可滚动布局，窗口恢复时会限制在当前屏幕可用范围；设置字段在 macOS、Windows 和 Linux 上均可随卡片宽度伸展，窄窗口自动换行。
- 任务快捷键仅在任务主页且对应操作可用时生效，避免在设置页误启动下载或覆盖未保存内容；空闲、运行和失败恢复状态只显示当前可用操作。
- 停止和退出改为后台终止 DCMTK 进程，界面不再同步等待；归档与 `dcmdump` 完整性校验可被取消，未处理文件继续保留在暂存目录。
- 诊断日志目录损坏或无权限时自动降级到安全的系统临时目录；极端情况下退回标准错误或内存日志，诊断初始化不再阻止程序启动。

## 2.6.3（2026-07-17）

- PDI 阅片入口统一为导出目录：界面只显示“打开影像”和“打开导出目录”，浏览器地址改用 `/viewer/directory/`，无需选择 JSON、`DICOMDIR` 或逐个影像文件。
- 保留隐藏的内部阅片索引作为大批量快速缓存，不在 PDI 根目录暴露 JSON；本地服务增加每次启动独立的会话保护、索引结构校验和 DICOM 文件白名单。
- 进一步限制本地服务只能读取当前 PDI 的 `DICOM` 子目录，拒绝符号链接、未被索引引用的文件和未授权的本机请求。
- 主程序打开旧版 PDI 时会自动使用旧阅片路由，升级后仍可读取原有 `DCMGET_STUDIES.json` 目录。

## 2.6.2（2026-07-16）

- PDI 改为以导出根目录为唯一用户入口：主界面提供“立即阅片”和“打开文件夹”，根目录不再暴露或要求用户选择 JSON，阅片元数据改由内部只读接口提供。
- 修复 Windows 无控制台模式下本地阅片服务可能因日志输出中断请求并出现 `ERR_EMPTY_RESPONSE` 的问题；启动前会校验离线 OHIF 的逐文件 SHA-256，损坏资源不会启动服务。
- 本地阅片服务只绑定 `127.0.0.1`，隐藏内部索引、拒绝未知 API 和路径越界；Windows 防火墙规则收紧为实际 `storescp.exe` 且仅允许域/专用网络。
- 阅片器会在本地服务和目录接口真正就绪后再打开浏览器，空闲退出延长到 4 小时；恢复元数据移入内部目录，超出 100,000 帧或 64 MiB 的阅片索引会提示拆分批次并保留可用的 DICOMDIR。
- 大批量恢复任务按需读取归档路径，超过 200 条仍保持摘要模式；失败任务新增“接受当前结果”，可直接使用已收到文件继续 PDI，CLI 同步支持 `--accept-download-failures`。
- 强化接收器存活监测、跨 association 文件归属、DICOM 完整性验证以及 SOP Instance UID 文件名、去重和冲突保护，降低并发接收时的部分失败和误归档风险。

## 2.6.1（2026-07-16）

- 修复 Windows 安装版和便携版已勾选 OHIF 后，因冻结程序内缺少可复制的本地服务脚本而错误导出为“不含 OHIF”的问题。
- Windows 冻结程序现在会同时携带 `OPEN_VIEWER.exe` 所需的独立服务和跨系统 Python 启动脚本，并在构建、自检及安装升级阶段验证资源完整性。
- `INDEX.HTM` 明确标注为检查清单而非阅片器，并优先提示 Windows 用户双击 `OPEN_VIEWER.exe`；缺少查看器时不再显示相互矛盾的说明。

## 2.6.0（2026-07-16）

- PDI 阅片改为直接使用导出的原始 DICOM 和 `DICOMDIR`，集成中文 OHIF 网页阅片器，不再转换 JPG，也不再携带 Weasis。
- PDI 阅片器资源和影像完全保存在便携目录中；通过只绑定 `127.0.0.1` 随机端口的本地只读 HTTP 启动器打开，阅片时不依赖互联网、外部 CDN 或在线服务。
- PDI 配置升级为 v5，旧 v4 配置的 PACS、AE、目录、机构名称和输出位置保留；旧 JPEG 与 Weasis 选项会合并迁移为 OHIF 开关，两项均关闭时继续保持关闭。
- PACS 端口和监听端口改为带范围校验的纯文本输入，避免鼠标滚轮误改；主页新增“打开目标目录”，并去除设置页重复的 DCMTK 就绪状态。

## 2.5.2（2026-07-16）

- 修复点击“开始下载”时 Qt 将按钮布尔状态误作检查号列表，导致 Windows 程序抛出 `TypeError: 'bool' object is not iterable` 的问题。
- 增加真实鼠标点击回归测试，同时保持失败重试和断点续传的参数传递逻辑不变。

## 2.5.1（2026-07-16）

- 移除启动时的每日日期口令；机器码注册码、30 次免费试用和恢复任务授权保持不变。
- 修复 macOS iCloud 目录中 Qt 平台插件被标记为 hidden 后导致的 `SIGABRT` 闪退，直接运行源码入口也会在加载 Qt 前自动修复插件标志。
- 新增固定诊断日志，覆盖启动、Python/Qt 未捕获异常、后台线程和原生崩溃；界面及 Windows 开始菜单均提供日志入口。
- 修复下载线程结束时过早释放 `QThread` 引用的竞态，避免任务完成、切换 PDI 或关闭窗口时发生原生退出。

## 2.5.0（2026-07-16）

- 新增按批次自动生成 PDI 便携目录：包含标准 `DICOMDIR`、静态 `INDEX.HTM`、原始 DICOM、网页预览、说明和 SHA-256 清单，可整体复制到 U 盘使用。
- PDI 只使用当前批次实际归档的文件；严格 Profile 不兼容时自动生成带明确警告的兼容目录，网页预览或查看器失败不会改变原下载结果。
- 设置页新增机构名称、PDI 输出目录、预览范围和 Windows Weasis 选项；导出过程支持独立进度、取消、打开目录和失败重试。
- Windows 安装版与独立目录版可内置固定为 4.7.1、经过 SHA-256 校验的 Weasis 便携查看器；源码部署脚本支持校验缓存并明确报告跳过或失败状态。
- 超过 200 个检查号时自动隐藏逐项任务表，仅显示总进度、当前检查号、累计文件数和状态汇总，避免数万行表格造成界面卡顿。
- 下载与 PDI 阶段按检查号写入跨平台 SQLite 恢复点；重启或意外退出后可跳过已处理项并继续剩余队列，复用已发布且校验通过的 PDI 目录，自动清理遗留 DCMTK 子进程，修正后的续传设置会写回恢复点，同一试用任务恢复不会重复扣次数。

## 2.4.0（2026-07-15）

- 下载核心新增连接与 DIMSE 超时、异常进程兜底清理，并降低速度统计对大检查的目录扫描压力。
- 界面明确区分等待暂停与已经暂停，折叠任务仍显示摘要，设置错误会直接显示并聚焦，重试进度按当前批次计算。
- 匿名密钥改为加锁后的原子创建；源码部署补齐构建依赖，CI 新增 Python 3.10 与真实 UI 启动检查。
- Windows 安装器、便携版和独立目录版拆分发布，避免为获取安装器下载三份重复运行时。

## 2.3.0（2026-07-15）

- 任务详情新增实际接收速度，下载中显示采样速度，完成后显示本检查号平均速度。
- 新增“暂停 / 继续”：当前检查号安全完成并归档后暂停队列，继续时从下一项接着下载，`storescp` 始终保持监听。
- 主界面新增“版本说明”入口，安装包和源码部署包同步携带完整更新记录。

## 2.2.0（2026-07-15）

- 新增基础脱敏、研究匿名和严格元数据匿名三种下载后处理方案。
- 匿名目录、文件名、UID 和日期可保持稳定映射；失败文件及日志保存到应用私有状态目录。
- 匿名输出采用临时写入、DICOM 校验和原子发布，风险内容不会被误标为已匿名。

## 2.1.0（2026-07-14）

- 新增每日日期口令、机器码注册码和 30 次免费试用。
- 新增常用目录组合模板，默认使用 `PatientID/AccessionNumber/StudyInstanceUID`，所有下载文件统一使用 `.dcm` 后缀。
- 默认本机 AE 调整为 `DCMGET`、接收端口调整为 `6666`。
- 新建任务区域支持收起；Windows 安装包支持原位升级并保留配置、注册码和试用记录。
- 加强 `storescp` 并发接收诊断及部分失败识别。

## 2.0.0（2026-07-13）

- 下载流程由 `movescu` 直接接收重构为常驻 `storescp` 接收、`movescu --no-port` 发起 C-MOVE。
- 支持 Windows x64、macOS ARM64/x86_64 和 Linux x86_64。
- 重构 PyQt5 界面、配置迁移、预检、日志、重试和跨平台进程清理。
- 新增跨系统部署脚本、源码部署包和四平台 CI 验证。

## 1.0.0（2023-12-08）

- 提供基于 `movescu` 的检查号批量下载、基础配置文件和初始桌面界面。
