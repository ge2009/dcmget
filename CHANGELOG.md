# DcmGet 版本说明

## 2.9.2（2026-07-17）

- 目标目录已有相同 SOP Instance UID 时保留原文件并跳过新收到的文件，不再把整项任务判为归档失败；重复信息合并为摘要，避免大量冲突逐行刷红色日志。
- 非匿名任务的下载与接收日志随影像写入目标目录 `_DcmGetLogs`；匿名任务日志、启动和崩溃诊断日志继续保留在应用私有状态目录，目标目录不可写时也会自动回退到实例日志目录。
- 任务结果区分新增、已存在跳过和失败数量，并提供打开影像目录、任务日志等快捷操作；输入区与详细日志默认收起，让进度和结果保持在主视区。
- 保持 PyQt5 和“一个窗口一个任务、应用可多开”的运行方式。自动发布暂时只生成 Windows x64 安装版、便携版和 ZIP，不构建 macOS/Linux 成品包。
- Windows 构建、源码部署和启动阶段统一拒绝 32 位运行时；发布前校验 Python、`DcmGet.exe`、`DcmGetPdiServer.exe`、`storescp.exe` 和 `movescu.exe` 均为 AMD64，PDI 的 Python 回退启动器也执行同一运行时校验。Inno Setup 6 的安装引导程序自身是 x86 兼容程序，但会拒绝 32 位 Windows，只安装 AMD64 应用载荷，并允许 Windows 11 ARM64 通过 x64 兼容层运行。

## 2.9.1（2026-07-17）

- 修复 Windows 安装版启动时，PyQt5 为 `aboutToQuit` 回调创建弱引用失败并报 `cannot create weak reference to 'InstanceProfile' object` 的问题；主窗口现在可正常打开。
- 加固退出清理顺序：即使界面状态通知或暂存目录清理发生异常，也会继续终止 `storescp`、`movescu` 并关闭任务日志，避免关闭程序后遗留 DCMTK 子进程。
- UI 自检现在走真实的实例配置分配和 Qt 退出回调连接；Windows 覆盖升级测试也会启动安装后的界面自检，防止该类仅在安装版出现的启动回归再次漏检。

## 2.9.0（2026-07-17）

- 移除单窗口多任务工作台、共享 pynetdicom 接收器池和并发调度器，恢复清晰的单窗口单任务流程；每个进程只运行一个下载任务，界面不再显示任务列表、全局日志范围或并发数量设置。
- DcmGet 桌面界面现在可以多开。每次启动自动取得持久实例槽 `i1`、`i2` 等，也可通过 `--profile N` 指定；配置、任务恢复点、QSettings 窗口偏好和日志目录按实例隔离，已被其他进程占用的槽不会重复打开。命令行继续作为单任务前台客户端并保留旧恢复参数兼容。
- 每个实例独立启动一个外部 DCMTK `storescp`，任务中的 `movescu` 按检查号顺序执行。不同实例必须配置不同监听端口，通常也应使用独立接收 AE，并在 PACS 中分别建立 Move Destination 映射。
- 当前 C-MOVE 接收时间窗内新到达的 DICOM 会直接归入当前检查号，不再因返回文件缺少、错误声明或使用不同的 `AccessionNumber` 而隔离；文件格式、SOP UID 和写入完整性校验仍保留。
- 每个检查号完成后立即写入当前实例恢复点；崩溃或重启后会优先重新取得有未完成任务的槽并继续，已完成项不重复下载。2.8.x `tasks.sqlite3` 中的未完成任务会一次性拆分迁移到实例槽，2.7.x 的全局 `active-task.sqlite3` 也会迁移；原数据库和迁移记录保留。
- 增加多进程安全保护：配置保存使用跨进程锁和独立临时文件；相同目标目录的 DICOM 发布使用跨进程锁并按内容去重，PDI 同秒发布会分配不同目录；每个进程使用独立诊断和崩溃日志文件。
- Windows 安装器和源码部署防火墙规则改为实际监听的 `storescp.exe`，仅限域/专用网络且不固定端口，可支持多个实例分别使用自定义接收端口。升级继续保留现有配置、注册码、试用状态和下载结果。
- Windows 单文件便携版首次启动会逐文件校验并将 DCMTK 原子发布到稳定的用户目录；多个便携实例同时启动会复用同一个 `storescp.exe` 路径，避免 PyInstaller 临时目录变化导致防火墙授权失效。
- `max_concurrent_moves` 仅为读取 2.8.x 配置兼容而保留，2.9.0 不再显示或生效；每个实例始终顺序执行 C-MOVE。

## 2.8.3（2026-07-17）

- 移除不同未完成任务之间的检查号重复拦截；新建任务、旧恢复点迁移和失败项重试不再弹出“检查号已存在于未完成任务中”，相同检查号可以写入各任务自己的目标目录。
- 同一任务内的重复输入仍按首次出现顺序自动去重；多个任务共享同一接收 AE 和端口时，调度器会避免相同检查号同时占用两个接收路由，防止并发串片。

## 2.8.2（2026-07-17）

- 多任务接收层升级为 C-STORE SCP 接收器池。不同端口会启动独立接收服务，并分别使用私有暂存目录、隔离目录和接收日志；相同 AE 与端口的任务复用一个支持并发 association 的接收器，各自写入独立任务目录。
- 未完成任务不再锁定 PACS 地址、PACS AE、本机调用 AE、接收 AE 和接收端口；每个任务继续使用创建时的完整配置快照，新设置只影响之后创建的任务。
- 增加接收端口保护：同一端口不能同时绑定不同接收 AE；同一 AE 可按 PACS 的 Move Destination 映射配置在不同端口启动多个接收器，端口冲突会在任务创建或接收器启动前给出明确修复提示。
- 保留应用级 DCMTK 路径与全局并发上限，多个接收器仍由统一调度器限制同时运行的 C-MOVE 数量，停止或退出时会完整清理各接收器和活动进程。

## 2.8.1（2026-07-17）

- 界面日志默认只显示错误；勾选“显示详细日志”后可查看调试、信息、成功和警告，磁盘任务日志与共享接收器日志始终完整保留。
- 任务列表支持安全删除已完成、失败、已取消及可重试的任务记录；删除不会移除已下载 DICOM、PDI、日志或隔离文件，运行、排队、暂停和 PDI 活动中的任务不能删除；迟到的后台更新也不会让已删除任务重新出现在列表。
- 修复下载完成后的延迟 Qt 信号可能重复触发 PDI 导出、把已经完成的 PDI 再次放回队列的竞态。
- 修复 Windows 接收暂存位于 `C:`、目标位于 `D:`、U 盘或网络共享时的跨卷归档失败；跨文件系统时改为目标侧安全复制、同步并原子发布，失败时保留源文件。
- 无法归属的接收文件、后台进程恢复信息保存失败及实时进度保存失败等关键异常改为错误级别，默认日志视图即可看到。
- 超过 200 条的任务在首次加载后直接使用增量摘要刷新，40,000 条任务不再因每完成一项都重复执行全表统计而造成界面周期性卡顿。
- PACS、本机调用与接收 AE Title 在预检阶段统一校验为 1-16 个可打印 ASCII 字符；中文、控制字符、反斜杠及超长输入会在对应设置字段直接提示，不再延迟到接收器或 `movescu` 启动后失败。

## 2.8.0（2026-07-17）

- 新增单窗口多任务工作台：左侧任务列表、右侧任务详情；可随时新建、暂停、继续、停止、重试并切换查看多个任务，窄窗口自动切换为整页列表/详情。
- 新增真正的多任务并发调度。全局共享一个跨平台 pynetdicom C-STORE SCP，默认同时运行 2 个 C-MOVE，可在 1-8 之间设置；空闲槽会立即接纳新任务，不必等待其他检查号完成。
- 每个活动检查号使用独立接收路由和暂存目录；优先按 Accession Number 精确归属，并以已绑定 Study Instance UID 兼容后续缺失检查号的对象。多个路由并行时无法确定归属的文件只进入隔离目录，不会按时间猜测或串入其他任务。
- 接收器支持全部标准 Storage SOP Class 和常见压缩传输语法，采用分块接收并保留原始编码落盘；Windows 安装器防火墙规则同步迁移到实际监听的 `DcmGet.exe`，源码部署规则指向虚拟环境 Python。
- 任务恢复库升级为 `tasks.sqlite3`，使用 WAL、短事务和逐条 SQL 调度；40,000 条检查号不会反复整体载入内存，旧 `active-task.sqlite3` 会安全迁移并保留备份。
- 多任务分别保存配置快照、进度、文件数、速度、日志、进程与 PDI 状态；超过 200 条时继续只显示聚合统计。未完成任务之间禁止重复检查号，无法归属的接收文件进入隔离目录。
- PDI 改为独立单任务队列，可与后续下载并行；PDI 失败可单独重试，不改变下载结果，也不额外消耗试用次数。
- 每个任务仅在第一次真正开始 C-MOVE 时扣一次试用次数；排队、暂停、恢复和重试不会重复扣次。
- 再次启动 DcmGet 会唤醒已有窗口，避免多个进程争抢接收 AE 和端口。退出时保留全部恢复点，下次启动一次性恢复所有未完成任务。
- CLI 新增 `--task-id` 指定恢复任务；存在多个未完成任务但未指定时会列出任务并退出，避免恢复错误任务。

## 2.7.1（2026-07-17）

- 修复 DICOM 错误声明为 `ISO_IR 100` 或其他单字节字符集时，PDI 检查卡中的 Study Description 和患者姓名中文乱码问题。
- 生成 PDI 阅片元数据时会忽略与显示无关的损坏私有文本标签；原始 DICOM 文件不会被修改。旧 PDI 目录需要重新导出后才能应用本次修复。

## 2.7.0（2026-07-17）

- 任务主页新增 PDI 快捷开关和输出目录选择，无需进入设置页即可快速启用并选择便携目录保存位置。
- PDI 中文 OHIF 阅片器默认单击选择序列，并默认显示患者四角标信息，优化日常阅片交互。
- 增加 DICOM 中文字符集容错和编码修复，改善患者姓名、检查与序列描述在 PDI 中的乱码问题。
- 换用全新 DcmGet 品牌图标，并优化高 DPI 窄窗口布局、大批量任务摘要以及阅片服务启动反馈。
- C-MOVE 结果会核对 PACS 完成子操作数与本地实际接收文件数，避免“PACS 报告完成但本地无文件”被误判成成功。
- `storescp` 接收器的 association abort 改为独立告警，不再无条件降级当前检查号状态；真实 DIMSE 失败仍会正确标记。
- 下载速度统计改为增量与自适应扫描，降低大量文件下频繁遍历目录的界面和磁盘开销。
- 研究匿名和严格匿名补充高风险自由文本清理，并覆盖嵌套序列中的敏感字段。

## 2.6.5（2026-07-17）

- 任务主页增加“打开已有 PDI 目录”：重启后可直接选择 U 盘或历史 PDI 根目录，程序记住上次位置，并只使用安装包内经过校验的 OHIF 与本地服务；新旧 PDI 都通过目录路由打开，用户不再需要寻找或选择 JSON 文件，也不会加载介质内可能被替换的网页脚本。
- 顶部状态与操作区改为稳定的双行布局，高 DPI 窄窗口不再挤压关键状态；禁用的主按钮会显示为明确的灰色状态，避免误判为可点击。
- C-MOVE 会解析最终 DIMSE 状态和完成、失败、警告子操作数；PACS 返回 `0xB000` 等部分失败时不再误报“完成”，而是保留已收文件并标记“部分成功”。
- PDI 取消改为后台幂等清理；Windows `taskkill` 增加超时和直接强制终止回退，停止或关闭时不再长时间卡住界面。
- DCMTK 部署缓存增加平台、版本、官方归档摘要及必需工具清单；每次复用都以固定 SHA-256 的 OFFIS 归档重新核对整个安装树，包含可执行文件、动态库、数据文件和链接，并拒绝缺失、篡改或额外注入的文件。Windows 构建同样要求保留并通过该官方归档认证。
- Windows 安装器将旧的固定 6666 防火墙规则迁移为仅限实际 `storescp.exe`、域/专用网络的程序规则，设置自定义接收端口后无需重新安装规则；升级仍保留原配置、注册码和试用状态。

## 2.6.4（2026-07-17）

- 主任务页增加高 DPI 可滚动布局，窗口恢复时会限制在当前屏幕可用范围；设置字段在 macOS、Windows 和 Linux 上均可随卡片宽度伸展，窄窗口自动换行。
- 任务快捷键仅在任务主页且对应操作可用时生效，避免在设置页误启动下载或覆盖未保存内容；空闲、运行和失败恢复状态只显示当前可用操作。
- 停止和退出改为后台终止 DCMTK 进程，界面不再同步等待；归档与 `dcmdump` 完整性校验可被取消，未处理文件继续保留在暂存目录。
- 诊断日志目录损坏或无权限时自动降级到安全的系统临时目录；极端情况下退回标准错误或内存日志，诊断初始化不再阻止程序启动。

## 2.6.3（2026-07-17）

- PDI 阅片入口统一为导出目录：界面只显示“打开影像”和“打开导出目录”，浏览器地址改用 `/viewer/directory/`，无需选择 JSON、`DICOMDIR` 或逐个影像文件。
- 保留隐藏的内部阅片索引作为大批量快速缓存，不在 PDI 根目录暴露 JSON；本地服务增加每次启动独立的会话保护、索引结构校验和 DICOM 文件白名单。
- 进一步限制本地服务只能读取当前 PDI 的 `DICOM` 子目录，拒绝符号链接、未被索引引用的文件和未授权的本机请求。
- 主程序打开旧版 PDI 时会自动使用旧阅片路由，升级后仍可读取原有 `DCMGET_STUDIES.json` 目录。

## 2.6.2（2026-07-16）

- PDI 改为以导出根目录为唯一用户入口：主界面提供“立即阅片”和“打开文件夹”，根目录不再暴露或要求用户选择 JSON，阅片元数据改由内部只读接口提供。
- 修复 Windows 无控制台模式下本地阅片服务可能因日志输出中断请求并出现 `ERR_EMPTY_RESPONSE` 的问题；启动前会校验离线 OHIF 的逐文件 SHA-256，损坏资源不会启动服务。
- 本地阅片服务只绑定 `127.0.0.1`，隐藏内部索引、拒绝未知 API 和路径越界；Windows 防火墙规则收紧为实际 `storescp.exe` 且仅允许域/专用网络。
- 阅片器会在本地服务和目录接口真正就绪后再打开浏览器，空闲退出延长到 4 小时；恢复元数据移入内部目录，超出 100,000 帧或 64 MiB 的阅片索引会提示拆分批次并保留可用的 DICOMDIR。
- 大批量恢复任务按需读取归档路径，超过 200 条仍保持摘要模式；失败任务新增“接受当前结果”，可直接使用已收到文件继续 PDI，CLI 同步支持 `--accept-download-failures`。
- 强化接收器存活监测、跨 association 文件归属、DICOM 完整性验证以及 SOP Instance UID 文件名、去重和冲突保护，降低并发接收时的部分失败和误归档风险。

## 2.6.1（2026-07-16）

- 修复 Windows 安装版和便携版已勾选 OHIF 后，因冻结程序内缺少可复制的本地服务脚本而错误导出为“不含 OHIF”的问题。
- Windows 冻结程序现在会同时携带 `OPEN_VIEWER.exe` 所需的独立服务和跨系统 Python 启动脚本，并在构建、自检及安装升级阶段验证资源完整性。
- `INDEX.HTM` 明确标注为检查清单而非阅片器，并优先提示 Windows 用户双击 `OPEN_VIEWER.exe`；缺少查看器时不再显示相互矛盾的说明。

## 2.6.0（2026-07-16）

- PDI 阅片改为直接使用导出的原始 DICOM 和 `DICOMDIR`，集成中文 OHIF 网页阅片器，不再转换 JPG，也不再携带 Weasis。
- PDI 阅片器资源和影像完全保存在便携目录中；通过只绑定 `127.0.0.1` 随机端口的本地只读 HTTP 启动器打开，阅片时不依赖互联网、外部 CDN 或在线服务。
- PDI 配置升级为 v5，旧 v4 配置的 PACS、AE、目录、机构名称和输出位置保留；旧 JPEG 与 Weasis 选项会合并迁移为 OHIF 开关，两项均关闭时继续保持关闭。
- PACS 端口和监听端口改为带范围校验的纯文本输入，避免鼠标滚轮误改；主页新增“打开目标目录”，并去除设置页重复的 DCMTK 就绪状态。

## 2.5.0（2026-07-16）

- 新增按批次自动生成 PDI 便携目录：包含标准 `DICOMDIR`、静态 `INDEX.HTM`、原始 DICOM、网页预览、说明和 SHA-256 清单，可整体复制到 U 盘使用。
- PDI 只使用当前批次实际归档的文件；严格 Profile 不兼容时自动生成带明确警告的兼容目录，网页预览或查看器失败不会改变原下载结果。
- 设置页新增机构名称、PDI 输出目录、预览范围和 Windows Weasis 选项；导出过程支持独立进度、取消、打开目录和失败重试。
- Windows 安装版与独立目录版可内置固定为 4.7.1、经过 SHA-256 校验的 Weasis 便携查看器；源码部署脚本支持校验缓存并明确报告跳过或失败状态。
- 超过 200 个检查号时自动隐藏逐项任务表，仅显示总进度、当前检查号、累计文件数和状态汇总，避免数万行表格造成界面卡顿。
- 下载与 PDI 阶段按检查号写入跨平台 SQLite 恢复点；重启或意外退出后可跳过已处理项并继续剩余队列，复用已发布且校验通过的 PDI 目录，自动清理遗留 DCMTK 子进程，修正后的续传设置会写回恢复点，同一试用任务恢复不会重复扣次数。

## 2.4.0（2026-07-15）

- 下载核心新增连接与 DIMSE 超时、异常进程兜底清理，并降低速度统计对大检查的目录扫描压力。
- 界面明确区分等待暂停与已经暂停，折叠任务仍显示摘要，设置错误会直接显示并聚焦，重试进度按当前批次计算。
- 匿名密钥改为加锁后的原子创建；源码部署补齐构建依赖，CI 新增 Python 3.10 与真实 UI 启动检查。
- Windows 安装器、便携版和独立目录版拆分发布，避免为获取安装器下载三份重复运行时。

## 2.3.0（2026-07-15）

- 任务详情新增实际接收速度，下载中显示采样速度，完成后显示本检查号平均速度。
- 新增“暂停 / 继续”：当前检查号安全完成并归档后暂停队列，继续时从下一项接着下载，`storescp` 始终保持监听。
- 主界面新增“版本说明”入口，安装包和源码部署包同步携带完整更新记录。

## 2.2.0（2026-07-15）

- 新增基础脱敏、研究匿名和严格元数据匿名三种下载后处理方案。
- 匿名目录、文件名、UID 和日期可保持稳定映射；失败文件及日志保存到应用私有状态目录。
- 匿名输出采用临时写入、DICOM 校验和原子发布，风险内容不会被误标为已匿名。

## 2.1.0（2026-07-14）

- 新增每日日期口令、机器码注册码和 30 次免费试用。
- 新增常用目录组合模板，默认使用 `PatientID/AccessionNumber/StudyInstanceUID`，所有下载文件统一使用 `.dcm` 后缀。
- 默认本机 AE 调整为 `DCMGET`、接收端口调整为 `6666`。
- 新建任务区域支持收起；Windows 安装包支持原位升级并保留配置、注册码和试用记录。
- 加强 `storescp` 并发接收诊断及部分失败识别。

## 2.0.0（2026-07-13）

- 下载流程由 `movescu` 直接接收重构为常驻 `storescp` 接收、`movescu --no-port` 发起 C-MOVE。
- 支持 Windows x64、macOS ARM64/x86_64 和 Linux x86_64。
- 重构 PyQt5 界面、配置迁移、预检、日志、重试和跨平台进程清理。
- 新增跨系统部署脚本、源码部署包和四平台 CI 验证。

## 1.0.0（2023-12-08）

- 提供基于 `movescu` 的检查号批量下载、基础配置文件和初始桌面界面。
